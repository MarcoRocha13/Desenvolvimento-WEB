Softaware de Versionamento

Sistema que tem por objetivo realizar o controle de versões de um código ou documento. É possível gerenciar as versões de um código ou documento realizados por um desenvolvedor ou equipe de desenvolvedores.

O software de versionamento irá gerenciar tanto o repósitorio local quanto o remoto.

O software de Versionamento pode ser:

Centralizado
--> Possui um repositório central em um servidor com controle de versões. Dependente do servidor, como Software Subversion.
Distribuido
--> Possui um repositório local com controle de versões. Atualizações no repositório central, é independente do servidor. Exemplo, Softaware Mercurial e Git

GIT

É um software livre de versionamento que surgiu para o desenvolvimento do Kernel do Linux, foi projetado e desenvolvido por Linus Torvalds em 2005.

Cada diretório de trabalho de Git é um repositório com um histórico completo e habilidade total de acompanhamento das revisões, não dependente de acesso a uma rede ou a um servidor central.

O softawe que será utilizado é o GitHub, que é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão. Surgiu em 2008 e foi adquirido pela Microsoft po UU$ 7,5 bilhões.
